# KI-Trainingsplan mit Mistral API

<RichText id="intro">

### Feature: KI-generierte Trainingsplane

Ersetze das statische Trainingsziel-Dropdown durch ein KI-gestutztes System: User beschreiben ihr Trainingsziel in natuerlicher Sprache, eine Supabase Edge Function ruft die Mistral API auf und gibt einen strukturierten JSON-Trainingsplan zurueck. Der Plan enthaelt 30 getaggte Ubungen + progressive Phasen. Jeden Tag werden 6 balanciert ausgewaehlt. Bestehende User behalten ihre alten predefined Plans bis sie neu konfigurieren.

**Konkreter Ablauf:** Ein neuer User oeffnet die App -> Tutorial -> waehlt "Kraft & Muskelaufbau" (Preset) -> Mistral generiert 30 Ubungen mit Tags (push/pull/legs/core/cardio/rest) + 4 progressive Phasen -> jeden Tag 6 balancierte Quests -> nach 2 Wochen auto-Progression zur naechsten Phase. In den Settings kann er jederzeit "Einrichten" klicken -> sieht seinen aktuellen Plan -> regeneriert mit eigenem Prompt ("Ich will mehr Fokus auf Beine").

</RichText>

<Callout id="key-decisions" tone="decision">

**Entschieden mit User:**
1. **API-Key**: Supabase Edge Function (`mistral-proxy`) haelt den Key server-side. Client ruft via `functions.invoke` auf.
2. **Ubungs-Pool**: Hybrid - KI darf bestehende nameKeys verwenden ODER neue Ubungen erfinden (mit DE-Texten direkt im JSON).
3. **30 Ubungen**: Flach mit Tags (push/pull/legs/core/cardio/rest). Balancing-Algorithmus waehlt 6/Tag so, dass verschiedene Tags vertreten sind.
4. **Rest Days**: Wochentag-Logik bleibt bestehen. KI taggt Ubungen mit `rest` - an Rest Days werden nur `rest`-getaggte Ubungen gewaehlt.
5. **Phasen**: KI generiert progressive Phasen (wie bestehendes System: weeks/sets/reps).
6. **Migration**: Dropdown verschwindet. Bestehende `goal`-Werte (muscle/endurance...) funktionieren weiter als predefined Plans. Erst wer "Einrichten" klickt und Custom Plan erstellt, wechselt.
7. **Tutorial**: 3 Presets (Kraft / Ausdauer / Abnehmen) + Custom-Textbox.
8. **Fallback**: Bei API-Fehler -> predefined Plan + Info-Popup "Fehler, Fallback-Plan genutzt".
9. **Regeneration-Limit**: 3x pro Tag (Counter in localStorage).

</Callout>

## Edge Function: mistral-proxy

<RichText id="edge-fn-heading">

### Supabase Edge Function Spezifikation

</RichText>

<Endpoint id="mistral-proxy-endpoint" method="POST" path="/mistral-proxy" summary="Generiert KI-Trainingsplan via Mistral API" auth="Supabase JWT (auto-attached)" params={[
  { name: "prompt", in: "body", type: "string", required: true, description: "User-Prompt oder Preset-Key ('kraft'/'ausdauer'/'abnehmen')" },
  { name: "userContext", in: "body", type: "object", required: false, description: "{ age, hasEquipment, difficulty, restDays, language }" }
]} request={{
  contentType: "application/json",
  example: '{ "prompt": "kraft", "userContext": { "age": 34, "hasEquipment": true, "difficulty": 3, "restDays": 2 } }'
}} responses={[
  { status: "200", description: "Validierter Trainingsplan", example: '{ "planName": "...", "exercises": [...], "stages": [...] }' },
  { status: "400", description: "Invalid prompt or JSON validation failed" },
  { status: "500", description: "Mistral API error or internal error" },
  { status: "429", description: "Rate limited" }
]}>

Die Edge Function:
1. Verifiziert JWT aus `Authorization` Header (Supabase Client haengt automatisch an).
2. Mapped Preset-Keys auf feste System-Prompts (`kraft` -> deutscher Prompt fuer Muskelaufbau).
3. Baut Mistral `chat/completions` Request mit `response_format: { type: "json_object" }`.
4. System-Prompt enthaelt die Liste aller verfuegbaren `nameKeys` aus `data/exercises.js` + JSON-Schema.
5. Validiert Response gegen Schema (30 exercises, tags, stages).
6. Gibt validiertes JSON zurueck.

</Endpoint>

<RichText id="edge-fn-deploy">

### Deployment

```bash
supabase functions deploy mistral-proxy
supabase secrets set MISTRAL_API_KEY=sk-...
```

Key bleibt server-side. Client sieht nur Responses. Keine Aenderung an `schema.sql` noetig (Custom Plans werden in IndexedDB gespeichert und via `app_data` synced).

</RichText>

  kraft: "Erstelle einen Trainingsplan fuer Kraft und Muskelaufbau mit Hanteln.",
  ausdauer: "Erstelle einen Trainingsplan fuer Ausdauer und Cardio-Training.",
  abnehmen: "Erstelle einen Trainingsplan fuer Gewichtsverlust und Fatburn."
};

const SYSTEM_PROMPT = \`Du bist ein Fitness-Experte. Erstelle einen Trainingsplan als JSON.
Erforderliche Felder:
- planName (string)
- planDescription (string)
- exercises (array von genau 30 Objekten)
- stages (array von 4 Phasen)

Jede Exercise braucht: nameKey, displayName, description, type (reps/time/check),
baseValue, tags (array), isRest (boolean), needsEquipment, muscles, statPoints, mana, gold.

Verwende diese bestehenden nameKeys wenn passend: [LISTE aus exercises.js]
Oder erfinde neue mit "custom_" Praefix.

Mindestens 4 exercises muessen isRest:true haben (fuer Rest Days).
Tags aus: push, pull, legs, core, cardio, rest, mobility, full_body.

response_format: json_object\`};

export default async (req: Request) => {
  const auth = req.headers.get("Authorization");
  if (!auth) return new Response("Unauthorized", { status: 401 });

  const { prompt, userContext } = await req.json();
  const userPrompt = PRESET_PROMPTS[prompt] || prompt;

  const response = await fetch(MISTRAL_URL, {
    method: "POST",
    headers: {
      "Authorization": \`Bearer \${MISTRAL_API_KEY}\`,
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      model: "mistral-small-latest",
      messages: [
        { role: "system", content: SYSTEM_PROMPT },
        { role: "user", content: userPrompt + JSON.stringify(userContext) }
      ],
      response_format: { type: "json_object" },
      temperature: 0.7
    })
  });

  const data = await response.json();
  const planJson = JSON.parse(data.choices[0].message.content);

  // Validierung: 30 exercises, tags vorhanden, stages vorhanden
  if (!planJson.exercises || planJson.exercises.length !== 30) {
    return new Response("Invalid plan: need 30 exercises", { status: 500 });
  }

  return new Response(JSON.stringify(planJson), {
    headers: { "Content-Type": "application/json" }
  });

## Balancing-Algorithmus

<RichText id="balancing-heading">

### 6-Quest-Auswahl aus 30 getaggten Ubungen

</RichText>

<AnnotatedCode id="balancing-code" filename="js/custom-plan-system.js" language="javascript" code={`const DQ_CUSTOM_PLAN = {
  // Waehlt 6 balancierte Ubungen aus 30 getaggten
  pickBalancedQuests(exercises, count = 6, isRestDay = false) {
    // 1. Filter: Rest Day -> nur isRest:true, sonst nur isRest:false
    let pool = isRestDay
      ? exercises.filter(ex => ex.isRest === true)
      : exercises.filter(ex => ex.isRest !== true);

    if (isRestDay) {
      // Rest Day: 5 zufaellige Rest-Uebungen (wie bestehendes System)
      return this.shuffle(pool).slice(0, 5);
    }

    // 2. Gruppiere nach primaerem Tag (erster Tag im tags-Array)
    const byTag = {};
    pool.forEach(ex => {
      const tag = ex.tags[0] || 'full_body';
      if (!byTag[tag]) byTag[tag] = [];
      byTag[tag].push(ex);
    });

    // 3. Wichtigste Tags first: push/pull/legs/core/cardio
    const priorityTags = ['push', 'pull', 'legs', 'core', 'cardio', 'full_body'];
    const selected = [];
    const usedKeys = new Set();

    // 4. Eine pro priorityTag bis 6 erreicht
    for (const tag of priorityTags) {
      if (selected.length >= count) break;
      const candidates = (byTag[tag] || [])
        .filter(ex => !usedKeys.has(ex.nameKey));
      if (candidates.length > 0) {
        const pick = candidates[Math.floor(Math.random() * candidates.length)];
        selected.push(pick);
        usedKeys.add(pick.nameKey);
      }
    }

    // 5. Falls weniger als 6: fuelle mit restlichen Ubungen auf
    while (selected.length < count && selected.length < pool.length) {
      const remaining = pool.filter(ex => !usedKeys.has(ex.nameKey));
      if (remaining.length === 0) break;
      const pick = remaining[Math.floor(Math.random() * remaining.length)];
      selected.push(pick);
      usedKeys.add(pick.nameKey);
    }

    // 6. recentExercises-Variety (wie bestehendes pickCandidate)
    // Bias gegen kuerzlich verwendete Ubungen
    return selected;
  },

  shuffle(arr) {
    const a = [...arr];
    for (let i = a.length - 1; i > 0; i--) {
      const j = Math.floor(Math.random() * (i + 1));
      [a[i], a[j]] = [a[j], a[i]];
    }
    return a;
  }
};`} annotations={[
  { lines: "5-6", label: "Rest Day Filter", note: "Wochentag-Logik in main.js entscheidet Rest Day. Hier: nur isRest Ubungen." },
  { lines: "11-16", label: "Gruppierung", note: "Ubungen werden nach primaerem Tag gruppiert." },
  { lines: "18-26", label: "Priority Selection", note: "Eine Ubung pro Tag-Kategorie -> balancierter Workout." },
  { lines: "28-34", label: "Auffuellen", note: "Falls weniger als 6 Kategorien: zufaellige Rest-Uebungen." }

<AnnotatedCode id="validation-code" filename="js/mistral-client.js" language="javascript" code={`const DQ_MISTRAL = {
    validatePlan(plan) {
        const errors = [];

        // 1. Top-Level Felder
        if (!plan.planName || typeof plan.planName !== 'string') errors.push('planName fehlt');
        if (!plan.exercises || !Array.isArray(plan.exercises)) errors.push('exercises fehlt');
        if (!plan.stages || !Array.isArray(plan.stages)) errors.push('stages fehlt');

        // 2. Genau 30 Ubungen
        if (plan.exercises && plan.exercises.length !== 30) {
            errors.push(\`Brauche 30 exercises, got \${plan.exercises?.length}\`);
        }

        // 3. Mindestens 4 isRest Ubungen
        if (plan.exercises) {
            const restCount = plan.exercises.filter(ex => ex.isRest === true).length;
            if (restCount < 4) errors.push('Mindestens 4 isRest Ubungen noetig');
        }

        // 4. Jede Ubung validieren
        const validTypes = ['reps', 'time', 'check', 'focus'];
        const validTags = ['push', 'pull', 'legs', 'core', 'cardio', 'rest', 'mobility', 'full_body'];
        const existingKeys = new Set(Object.values(DQ_DATA.exercisePool).flat().map(ex => ex.nameKey));

        if (plan.exercises) {
            plan.exercises.forEach((ex, i) => {
                if (!ex.nameKey || typeof ex.nameKey !== 'string') errors.push(\`exercise[\${i}].nameKey fehlt\`);
                if (!validTypes.includes(ex.type)) errors.push(\`exercise[\${i}].type invalid: \${ex.type}\`);
                if (!ex.tags || !Array.isArray(ex.tags) || ex.tags.length === 0) errors.push(\`exercise[\${i}].tags fehlt\`);
                if (typeof ex.baseValue !== 'number' || ex.baseValue < 1) errors.push(\`exercise[\${i}].baseValue invalid\`);
                if (!ex.displayName) errors.push(\`exercise[\${i}].displayName fehlt\`);

                // Hybrid: bestehende nameKeys OK, neue muessen custom_ haben
                if (!existingKeys.has(ex.nameKey) && !ex.nameKey.startsWith('custom_')) {
                    errors.push(\`exercise[\${i}].nameKey muss custom_ Praefix haben wenn neu\`);
                }
            });
        }

        // 5. Stages validieren
        if (plan.stages) {
            if (plan.stages.length < 2) errors.push('Mindestens 2 Phasen noetig');
            plan.stages.forEach((stage, i) => {
                if (typeof stage.weeks !== 'number') errors.push(\`stage[\${i}].weeks fehlt\`);
                if (typeof stage.sets !== 'number') errors.push(\`stage[\${i}].sets fehlt\`);
                if (typeof stage.reps !== 'number') errors.push(\`stage[\${i}].reps fehlt\`);
            });
            // Letzte Stage muss infinite sein
            const last = plan.stages[plan.stages.length - 1];
            if (last && last.weeks < 9999) errors.push('Letzte Phase muss weeks:9999 haben');
        }

        return { valid: errors.length === 0, errors };
    },

    async generatePlan(prompt, userContext) {
        try {
            const { data, error } = await DQ_SUPABASE.client.functions.invoke('mistral-proxy', {
                body: { prompt, userContext }
            });
            if (error) throw error;

            const validation = this.validatePlan(data);
            if (!validation.valid) {
                console.error('Plan validation failed:', validation.errors);
                throw new Error('Invalid plan: ' + validation.errors.join(', '));
            }

            return data;
        } catch (e) {
            console.error('Mistral API error:', e);
            DQ_UI.showCustomPopup(
                'Fehler bei Plan-Generierung. Ein Fallback-Plan wird genutzt.',
                'penalty'
            );
            throw e; // Caller entscheidet Fallback
        }
    },

    getRegenerationCount() {
        const today = DQ_CONFIG.getTodayString();
        const data = localStorage.getItem('dq_mistral_regen');
        if (!data) return 0;
        try {
            const parsed = JSON.parse(data);
            return parsed.date === today ? parsed.count : 0;
        } catch { return 0; }
    },

    incrementRegeneration() {
        const today = DQ_CONFIG.getTodayString();
        const count = this.getRegenerationCount();
        localStorage.setItem('dq_mistral_regen', JSON.stringify({ date: today, count: count + 1 }));
    },

    canRegenerate() {
        return this.getRegenerationCount() < 3;
    }
};`} annotations={[
